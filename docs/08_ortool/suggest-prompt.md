# 开源优化算法与传统机器学习仓库清单（分级分类+全维度画像）
## 评分说明
整体评分采用**10分制**，核心参考维度：收藏数/流行度（40%）、社区认可度（30%）、功能完整性（20%）、泛化性（10%）；评分越高代表综合实用性越强，优先推荐选择。

---

## 一、组合优化/数学规划/元启发式优化类
### 核心仓库（按评分从高到低排序）
#### 1. google/or-tools
| 基础画像 | 详情 |
|----------|------|
| 创建时间 | 2010年 |
| 最后维护时间 | 2026年（持续活跃） |
| 收藏数/流行度 | GitHub星标≈30k+，工业界标杆级优化库，Google官方维护，社区生态极完善 |
| 认可度 | 全球运筹学/组合优化领域首选工业级工具，被亚马逊、UPS等企业广泛用于物流/调度场景 |
| 编程语言 | 核心C++，提供Python/C++/Java/.NET接口 |

**功能详解**：
- 核心功能：工业级组合优化套件，内置CP‑SAT（约束满足）、GLOP（线性规划）、CBC（整数规划）等求解器；原生支持图优化、网络流计算，集成TSP/VRP专用求解逻辑，支持多约束场景的工程化落地。
- 使用场景：
  - 物流领域：带时间窗/容量约束的车辆路径规划（CVRPTW）、跨区域配送网络优化；
  - 生产领域：工厂设备排班、多工序生产调度、物料分配优化；
  - 人力领域：员工多技能排班（含休息/加班约束）、客服坐席调度；
  - 通用场景：旅行商问题（TSP）、任务分配、网络流资源分配。
- 解决的问题：工业级复杂约束下的组合优化问题，替代手工规划/简易算法导致的成本高、效率低问题；
- 能力边界：擅长离散型组合优化（整数/约束规划），对超高维非线性规划支持较弱；
- 泛化性：支持多语言调用，可集成到Java/C++/Python主流工程框架，适配Windows/Linux/macOS，支持分布式部署。
- 综合评分：9.8

#### 2. Pyomo/pyomo
| 基础画像 | 详情 |
|----------|------|
| 创建时间 | 2008年 |
| 最后维护时间 | 2026年（持续活跃） |
| 收藏数/流行度 | GitHub星标≈6k+，Python数学规划建模事实标准，学术/工业界双认可 |
| 认可度 | 被斯坦福、MIT等高校用作运筹学教学工具，电力/供应链领域头部企业广泛使用 |
| 编程语言 | Python |

**功能详解**：
- 核心功能：通用数学规划建模框架，支持线性规划（LP）、混合整数规划（MIP）、非线性规划（NLP）、随机规划；可无缝对接CBC、GLPK、Gurobi、CPLEX等第三方求解器，语法贴近数学表达式，建模灵活。
- 使用场景：
  - 能源领域：电力机组组合优化、电网调度、新能源消纳规划；
  - 供应链领域：仓库选址、配送路径联合优化、批量采购规划；
  - 金融领域：投资组合优化（含风险约束）、资产配置规划；
  - 工程领域：化工流程参数优化、机械设计参数寻优。
- 解决的问题：统一各类数学规划问题的建模范式，避免针对不同求解器重复编写代码；
- 能力边界：仅负责建模，需依赖第三方求解器完成计算，无原生并行计算支持；
- 泛化性：纯Python实现，易集成到数据分析流水线（Pandas/NumPy），支持自定义求解器接口。
- 综合评分：9.5

## 整体总结
### 核心推荐优先级
1. **组合优化领域**：优先选择google/or-tools（工业级全功能）、Pyomo/pyomo（通用建模）、casadi/casadi（非线性+自动微分）；
2. **传统ML领域**：优先选择scikit-learn（通用ML）、statsmodels（统计推断）、LIBSVM（轻量SVM）；
3. **学习/复用优先**：核心仓库配套的Demo仓库（如google/or-tools/docs、scikit-learn/scikit-learn.github.io）可直接复用，降低开发成本。

### 关键选型建议
1. 工业级复杂约束优化→google/or-tools；
2. 通用数学规划建模→Pyomo/pyomo；
3. 非线性优化+自动微分→casadi/casadi；
4. 传统ML快速落地→scikit-learn；
5. 统计推断分类→statsmodels；
6. 轻量SVM求解→LIBSVM；
7. 超参数优化→optuna/optuna；
8. 教学/小场景元启发式算法→guofei9987/scikit-opt。

### 能力边界避坑
1. 非凸优化问题避免使用cvxpy（仅支持凸优化）；
2. 大规模深度学习优化优先选择专业框架（如PyTorch），而非facebookresearch/optimizers；
3. 工业级场景避免使用已停更库（如Optunity）；
4. 高维连续无梯度优化优先选择CMA-ES/pycma，而非通用元启发式算法。
